# Sundial Editor CLI

`@arcridge/sundial-editor-cli` is the local provider bridge used by Sundial Editor. It runs on Node.js 20 or newer and supports a locally authenticated Codex CLI `0.131.0` or newer when that installation passes Sundial's app-server capability probe. Newer Codex versions are not rejected solely because their version differs from the build used to generate earlier protocol schemas.

`sundial-editor-cli health --provider codex` reports the exact Codex executable path and version resolved from the CLI process's `PATH`. It initializes app-server, validates model discovery and the required thread/history operations, checks turn RPC recognition with invalid parameters, and archives its temporary probe thread. Successful results are cached for 24 hours in machine-local configuration and invalidated when either the Sundial CLI or Codex version changes; pass `--refresh` to force a new probe. The probe never starts a model turn. A missing or malformed required capability is reported by RPC name with an update/retry diagnostic.

Sundial queries `model/list` before starting a thread or turn. An explicit Sundial model request must match the visible catalog and is passed to Codex. Without an explicit request, Sundial omits the model override—even when one catalog row has `isDefault: true`—so Codex applies its configured model and resumed threads retain their existing model.

The package installs two executables at the same version:

- `sundial-editor-cli` is the editor control surface. It manages source annotations, named agents and their queues, sessions, transcripts, and lifecycle controls. Run `sundial-editor-cli help` for its complete command surface.
- `sundial-agent-tools` is the intentionally narrow surface described to managed agents. Run `sundial-agent-tools help` to see the agent-facing surface.

Managed agents inspect each other's current coordination state and structured file claims with `coordination list`, then publish `working`, `waiting`, `blocked`, or `stopped` updates with `coordination update`. They publish assignment progress with `provide-status-update` and may write optional Markdown code annotations against any useful workspace file and line with `annotate`. To finish an assignment, the agent writes its complete Markdown answer to the announced `.sundial/<UserAnnotationId>response.md` handoff and invokes `record-task-response` with that exact one path. The command durably appends the official response to its originating user annotation, completes the assigned work, consumes the handoff, and returns only the normalized source path. Ordinary provider output is not treated as an official response.

The editor executable accepts structured JSON for its machine commands and exposes health, annotation, agent, work-queue, transcript, open, interrupt, reset, and Git workflow operations. Its annotation surface can re-anchor one saved source deterministically while preserving stable identities and reciprocal links. The workflow surface reads and moves a first-parent diff baseline, creates current-file or all-file temporary checkpoints, consolidates the temporary suffix and dirty work into a permanent commit, and repairs annotation companions and their links for Git-reported source moves and deletes. Git is invoked with structured arguments, and mutations reject unresolved conflicts, in-progress operations, unsafe temporary stacks, and any temporary commit reachable from `origin`.

Temporary commits use the exact complete message `Sundial:temp` and are local-only iteration state. Current-file checkpoints include a dirty companion for the selected source. Consolidation preserves companion bytes and stable annotation identities while replacing the verified temporary suffix with the requested permanent commit.

Annotation commands store linked user and agent source annotations in compact companions under the workspace's mirrored `.sundial/` tree. Each annotation records its immutable permanent-base commit. Reads return the CLI-resolved current permanent commit and exact annotation-ID membership used by the editor filter. Malformed or obsolete companions remain untouched rather than being guessed into the current format.

Named agents, queued work, replaceable sessions, ordered status histories, and per-session coordination histories persist separately under `.sundial/agents/`. This runtime directory is CLI-owned and gitignored; it is not part of the checked-in annotation companions.
Agent listing, prompt targeting, and queue readiness use only this persisted state. They do not contact Codex; a missing provider thread is detected when an explicit provider operation attempts to read or resume it.
